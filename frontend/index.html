System Info

Architecture, stack, and technical overview of FraudShield

10,000+ Synthetic Transactions
~92–96% Model Accuracy
6 ML Features
6 REST Endpoints
TECH STACK
Data Synthetic dataset with realistic fraud patterns
ML Random Forest · Scikit-learn · SMOTE
API Python · Flask · REST · JWT
DB MySQL · Relational Schema
UI HTML · CSS · JavaScript
Auth JWT Tokens · bcrypt
FEATURE ENGINEERING
amount — transaction amount in ₹
hour_of_day — 24-hour format (0–23)
merchant_category — encoded category (0–9)
transaction_count_24h — recent activity
distance_from_home_km — location deviation
is_online — online vs offline transaction
ARCHITECTURE FLOW
User enters transaction details in UI
↓
Frontend sends JSON to Flask API
↓
Input mapped to ML feature vector
↓
StandardScaler normalizes features
↓
Random Forest Model predicts fraud
↓
Probability + Risk Level calculated
↓
Result stored in MySQL + shown in UI
SECURITY
JWT authentication with expiry handling
bcrypt password hashing
Role-based access control
Input validation on all endpoints
